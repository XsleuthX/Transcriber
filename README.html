Quick start

1
Import a source. Use Import Video, YouTube, or Google Drive.
2
Choose language. Use the language dropdown above the workflow row. For mixed English/Chinese, use Auto Detect.
3
Open Transcribe. Confirm Model / Device / Compute, then click Start Transcribe.
4
Edit and review. Use SRT mode for cue-by-cue edits or TXT mode for a full text editing view.
5
Export. Use Export ▾ to save SRT or VTT.

Import sources

Local video/audio

Use Import Video. The app caches a 16 kHz mono WAV on the backend so Transcribe and Align-To-Audio can reuse it without re-uploading the same media repeatedly.

Import SRT/VTT

Use Import SRT/VTT to load existing captions into Sub A. In Dual Sub mode, Sub A is usually the original and Sub B is usually the translation.

YouTube

Use the YouTube icon. You can preview, import closed captions, download authorized videos, or transcribe by extracting temporary audio through the backend.

Use only with videos you own or have permission to process.

Google Drive

Use the Drive icon for shared Drive files. Preview + cache audio gives the best playback sync, because the app can use a native cached media preview instead of the Drive iframe.

Large uploads: If users access your app through ngrok, local videos must upload from their browser to your backend computer before transcription begins. YouTube/Drive sources avoid that large browser upload.

Transcribe

Open Transcribe. Model / Device / Compute are grouped in one row. Device defaults to cuda for GPU use; switch to CPU if CUDA is unavailable.

  • Language: Use Auto Detect for mixed English/Chinese. Select a fixed language when the content is known.
  • Model: Large models are more accurate but slower.
  • Progress: The compact progress strip appears only during backend jobs. The Cancel button appears only when a job is active.

Align subtitles

Align-To-Audio

Paste SRT or subtitle lines into the Align drawer and run Align-To-Audio. The backend analyzes audio and returns timed cues. This is useful when your text exists but timing is wrong.

Align-To-SRT

Maps pasted lines onto the timing of Sub A, creating or updating Sub B. This is useful for translations that should follow existing timing.

Translate & check

Open Translate for translation and review tools.

  • Start Translate: Translate Sub A into Sub B using the configured translation backend.
  • Dictionary: Store preferred term mappings such as names, organizations, and technical terms.
  • AI Check: Compare Sub A and Sub B. Semantic mode checks meaning. Anchor Drift mode checks numbers, acronyms, and key terms.
  • Key Terms: Appears when Anchor Drift is selected.

AI Assistant

The AI Assistant drawer gives you a dedicated prompt and answer panel. Choose context such as Sub A, Sub B, both subtitles, or the current cue.

Useful prompts include:

  • Summarize this transcript in five bullet points.
  • Find the strongest quote in the current section.
  • Rewrite the selected cue in documentary tone.
  • Suggest chapter breaks and titles.

DeepSeek features require your backend to have DEEPSEEK_API_KEY set.

View, captions, and timecode

Large timecode bar

The large timecode bar stays visible for frame/time checking. FPS can be changed from the top toolbar.

Settings

The compact Settings tab contains Source TC, View Mode, Captions, and Tips. These stay collapsed until needed.

TXT mode

TXT mode shows transcript text in an editable big box. Edits update the cue text line-by-line. Clicking a TXT line seeks to that cue's start time.

Captions

Caption settings include font, size, color, and maximum characters per subtitle row for overlay wrapping.

Source TC

Use Source TC to display/export captions with source timecode offset. The app can attempt to read source timecode from media metadata when available.

Share & collaborate

Use Share to create session links.

  • View-only: Others can review but not edit.
  • Editor: Opens the same snapshot with editing controls enabled.
  • Collaborative: Creates a live polling session that syncs changes across users every few seconds.

Local uploaded video files are not automatically shared to viewers. Web sources such as YouTube or Google Drive are better for remote review.

Backend notes

The backend pill shows which backend the app is connected to. For ngrok testing, make sure the frontend uses same-origin API:

const API_BASE = window.API_BASE || window.location.origin;

Run the backend with CUDA auto-language support if available:

set "WHISPER_AUTO_LANGS=en,zh"
python -m uvicorn server_whisper_deepseekv1:app --host 127.0.0.1 --port 8000
